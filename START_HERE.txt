🆘 COMMON ISSUES:

Problem: "Docker is not installed"
Solution: Download from https://www.docker.com/products/docker-desktop
          Run installer, restart computer, test with: docker --version

Problem: "Port 3001 is already in use"
Solution: Edit docker-compose.yml
          Change: "3001:3001" to "3002:3001"
          Then open: http://localhost:3002

Problem: "Cannot connect to MongoDB"
Solution: 1. Check .env file has correct MONGODB_URI
          2. Verify your internet connection
          3. MongoDB is in the cloud, needs internet access

Problem: "App shows blank page"
Solution: 1. Wait 5-10 seconds for server to fully start
          2. Refresh browser (Ctrl+R)
          3. Check logs: docker-compose logs backend

Problem: "kubectl command not found"
Solution: Install Kubernetes cluster or kubectl
          Your cluster setup provides kubectl

Problem: "Kubernetes cluster not found"
Solution: 1. Make sure your cluster is running
          2. Test with: kubectl cluster-info
          3. If error, cluster is not started yet

🎯 WHAT YOUR APP DOES:

When Running Locally:
✓ Frontend (index.html) loaded at http://localhost:3001
✓ Backend (server.js) provides API endpoints
✓ Data stored in MongoDB Cloud (Atlas)
✓ User authentication with JWT tokens
✓ Create, read, update, delete to-do items
✓ Organize tasks in lists (My Day, Work, Personal, etc.)
✓ Mark tasks as important or completed
✓ Set due dates and search tasks

When Running on Kubernetes:
✓ 2 pod replicas for redundancy
✓ Auto-scaling: grows to 5 pods when busy, shrinks to 2 when idle
✓ Load balancing across pods
✓ High availability: app keeps running even if one pod fails
✓ Same features as local version

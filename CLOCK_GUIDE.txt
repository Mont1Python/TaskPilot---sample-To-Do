================================================================================

What is this?

A beautiful, real-time clock dashboard that shows:
  ✓ Digital clock (HH:MM:SS) - updates every 100ms
  ✓ Analog clock with smooth second hand
  ✓ Hour, minute, second breakdown
  ✓ Current date and day
  ✓ Beautiful gradient UI with animations
  ✓ Deployed on Kubernetes with Ansible

================================================================================

📋 FILES CREATED:

1. clock.html
   - Standalone HTML file with beautiful clock UI
   - Can be opened directly in browser
   - Real-time updates every 100ms

2. deploy-clock.yml
   - Ansible playbook to deploy clock to Kubernetes
   - Creates namespace, deployment, service
   - Configures ConfigMaps with HTML content

3. deploy-clock.ps1
   - PowerShell script to run Ansible playbook
   - One command to deploy everything
   - Automatically checks prerequisites

    ✓ Exposes on port 80

🛑 STOPPING/REMOVING CLOCK:

Delete everything:
  kubectl delete namespace clock-dashboard

Delete just deployment:
  kubectl delete deployment live-clock -n clock-dashboard

Delete just service:
  kubectl delete svc live-clock-service -n clock-dashboard

❌ TROUBLESHOOTING:

Clock not showing?
  - Check if HTML file is loading: Inspect browser console (F12)
  - Ensure JavaScript is enabled
  - Try refreshing page

Ansible deployment fails?
  - Install Ansible: pip install ansible-core kubernetes
  - Check kubectl: kubectl cluster-info
  - Verify cluster is running: kubectl get nodes

Service shows "Pending":
  - For cloud clusters: Wait 5-10 minutes for LoadBalancer IP
  - For local clusters: Use NodePort or port-forward
  - Check: kubectl get svc -n clock-dashboard

Pod not running?
  - Check pod status: kubectl get pods -n clock-dashboard
  - View pod logs: kubectl logs <pod-name> -n clock-dashboard
  - Describe pod: kubectl describe pod <pod-name> -n clock-dashboard

Can't access from browser?
  - Verify LoadBalancer IP is assigned
  - Check firewall allows traffic on port 80
  - Try: kubectl port-forward svc/live-clock-service 8080:80 -n clock-dashboard
  - Then access: http://localhost:8080

📊 ANSIBLE PLAYBOOK DETAILS:

What the playbook does:

  1. Creates namespace "clock-dashboard"
  2. Creates ConfigMap with clock configuration
  3. Creates ConfigMap with HTML content
  4. Creates Deployment with:
     - Nginx Alpine image
     - 1 replica
     - Health probes
     - Resource limits
  5. Creates LoadBalancer Service
  6. Waits for service to be ready
  7. Displays deployment information

Playbook Variables:
  - clock_namespace: "clock-dashboard"
  - clock_app_name: "live-clock"
  - docker_image: "clock-app:latest"

Requirements:
  - ansible-core
  - kubernetes Python module
  - kubectl configured

Run with verbose:
  ansible-playbook deploy-clock.yml -v

Run with debug:
  ansible-playbook deploy-clock.yml -vvv
